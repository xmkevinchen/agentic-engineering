# Feature: Unified Output Conventions Implementation

## Goal

Make the file output paths, naming, formats, and temporary persistence behavior fully consistent across all ae skills, so that agents only need to read pipeline.yml to know where to write.

## Steps

- [x] Rewrite `plugins/ae/templates/pipeline.template.yml` — update output block to 6 semantic slots + defaults, add scratch config

## Acceptance Criteria

### AC1: pipeline.yml semantic slots complete
Template includes 6 output slots (discussions, plans, milestones, backlog, reviews, analyses) + scratch config. Each slot has a default value comment. pipeline.yml generated by `/ae:setup` matches this template.

### AC2: Zero-config usable (revised)
~~Running `/ae:plan test-feature` in an empty project (no pipeline.yml) causes the skill to write `docs/plans/001-test-feature.md` using defaults. No errors, no need to run setup first.~~

Revised: When no pipeline.yml exists, automatically trigger `/ae:setup` flow (informing the user "First time using ae plugin, initializing..."), then continue executing the original command after setup completes. User doesn't need to manually run setup, but pipeline.yml always exists.

### AC3: All file-writing skills read pipeline.yml
Check each of the 7 file-writing skills (analyze, discuss, plan, work, review, think, setup) and confirm each reads paths from `pipeline.yml → output.<slot>` with no hardcoded paths. Naming format is uniformly `NNN-slug` + frontmatter.

### AC4: High-value skills prompt save on completion
After running `/ae:trace`, `/ae:consensus`, or `/ae:think`, the agent uses AskUserQuestion to ask the user whether to formally save. User says yes → file is moved from scratch to the formal directory.

### AC5: Pipeline skills auto-write to scratch
After running `/ae:code-review`, verify the scratch directory contains an action log file with findings + action + status fields.

### AC6: Session recovery
Manually create a `status: in_progress` action log in scratch, then run any ae skill — the agent should proactively remind the user there are unfinished operations.

### AC7: cross-family-review moved out of skills
`plugins/ae/skills/cross-family-review/` directory does not exist. Content moved to `plugins/ae/docs/`. README shows 12 skills.
